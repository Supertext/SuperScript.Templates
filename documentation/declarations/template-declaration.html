The Template Declaration

SuperScript.Templates contains a single implementation of DeclarationBase, TemplateDeclaration. 

    public class TemplateDeclaration : SuperScript.Declarables.DeclarationBase
    {
        string Path
        string Template { get; set; }
    }

This class can accept either the contents or the path of an HTML template.

Name Type Description
Path string

Gets or sets the path to a file containing the template markup.

The setter will retrieve the template markup from the specified path and assign this to HTML template.

The setter will also set the DeclarationBase.Name property from the specified file name (without file extension).

Template string

Gets or sets the template markup.

Inherited members
Comment string

This text may be emitted onto its own line or after a variable, depending upon the specific implementation of DeclarationBase.

Optional.

EmitterKey string

Indicates which instance of IEmitter the content should be added to.

Optional. If not specified then the contents will be added to the default implementation of IEmitter.

Read more about the default emitter here.

Name string

Gets or sets the name assigned to this declarable.

Required.

Example

Instances of TemplateDeclaration can be declared as in the following example. 

    TemplateDeclaration declTemplate = new SuperScript.Templates.Declarables.TemplateDeclaration();
    declTemplate.Name = "templateNames";
    declTemplate.Template = "<li>${Name}</li>";
    declTemplate.EmitterKey = "templates";
    
    SuperScript.Declarations.AddDeclaration<TemplateDeclaration>(declTemplate);

The emitted output is dependent upon the modifiers in the referenced IEmitter. Therefore this example will go no further, but TemplateStringifyHtml and TemplateStringifyJavaScript both contain more complete examples.
